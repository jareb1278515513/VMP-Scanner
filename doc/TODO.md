## M2：漏洞探测层（优先级 P0）

### 2.1 插件框架

- [ ] 设计插件基类（metadata/match/probe/verify/evidence）
- [ ] 实现插件注册与加载机制（静态注册或自动发现）
- [ ] 实现统一执行器（按目标分发插件）
- [ ] 支持测试模式与攻击模式开关
- [ ] 增加插件执行日志与异常隔离（单插件失败不影响全局）

交付标准：
- [ ] 至少 1 个示例插件可独立执行并输出 Finding

### 2.2 Payload 字典管理（参考 PayloadsAllTheThings）

- [ ] 建立 scanner/payloads 目录和分类字典文件
- [ ] 按漏洞分类维护字典（sqli/xss/csrf/path_traversal）
- [ ] 为每个 payload 增加元数据（用途、风险等级、预期特征）
- [ ] 区分测试模式与攻击模式 payload
- [ ] 增加字典版本号和更新记录

交付标准：
- [ ] 插件可按漏洞类型加载对应 payload 集合
- [ ] 高风险 payload 默认禁用，需显式开启

### 2.3 首批漏洞插件

- [ ] SQL 注入检测插件（错误回显/布尔差异基础验证）
- [ ] XSS 检测插件（反射型回显检测）
- [ ] 敏感路径探测插件（字典枚举 + 状态判定）
- [ ] CSRF 基础检测插件（表单 token 缺失检查）
- [ ] 弱口令策略评估插件（限速、低频、可配置）

交付标准：
- [ ] 至少 3 个插件在 DVWA 场景可稳定输出结果
- [ ] 每个插件都包含 evidence 字段用于复核
